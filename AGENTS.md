# AGENTS.md

이 문서는 이 저장소에서 작업하는 에이전트가 따라야 할 기본 규칙을 정리합니다.

## 커밋 원칙

- 커밋 메시지는 한국어로 작성합니다.
- 하나의 커밋에는 하나의 의도만 담습니다.
- 기능 추가, 버그 수정, 리팩터링, 문서 변경처럼 성격이 다른 작업은 커밋을 분리합니다.
- 같은 목적의 변경은 파일이 여러 개여도 하나의 커밋으로 묶습니다.
- 사용자가 직접 만든 변경사항은 되돌리거나 덮어쓰지 않습니다.
- 커밋 전에는 `git status`와 `git diff`로 포함될 변경 범위를 확인합니다.

## 커밋 메시지 작성 규칙

- 제목은 변경의 핵심 목적을 짧고 구체적으로 작성합니다.
- 제목만 보고도 무엇이 바뀌었는지 알 수 있어야 합니다.
- 본문에는 실제 변경 내용, 변경 이유, 영향 범위를 필요한 만큼만 적습니다.
- 변경하지 않은 파일이나 관련 없는 작업은 언급하지 않습니다.
- 단순 포맷팅, 오타 수정처럼 영향이 작은 변경은 본문을 생략할 수 있습니다.
- 의미 있는 검증을 수행했다면 본문에 결과를 남깁니다.
- 검증하지 못했다면 그 이유를 커밋 본문이나 작업 요약에 명확히 남깁니다.

## 권장 형식

```text
커밋 제목

- 변경 내용:
  - 실제로 수정한 범위를 구체적으로 작성
- 변경 이유:
  - 이 변경이 필요한 이유나 사용자에게 미치는 영향 작성
- 검증:
  - 실행한 테스트, 수동 확인, 또는 검증하지 못한 이유 작성
```

## 예시

```text
동영상 속도 패널 표시 방식 개선

- 변경 내용:
  - 콘텐츠 스크립트에서 속도 패널 렌더링 위치를 조정
  - 팝업 스타일과 패널 간격을 정리
- 변경 이유:
  - 사이트별 레이아웃과 겹치는 문제를 줄이고 가독성을 개선
- 검증:
  - YouTube와 Netflix 화면에서 패널 표시 동작을 수동 확인
```
