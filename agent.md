# Agent Rules

## 커밋 메시지 규칙

- 커밋 메시지는 반드시 한국어로 작성합니다.
- 제목은 커밋의 핵심 목적을 짧게 요약합니다.
- 본문은 실제 변경 내용을 변경 범위별로 세분화해서 작성합니다.
- 성격이 같은 변경은 가능한 한 하나의 커밋으로 묶어 정리합니다.
- 기능 추가, 디자인 변경, 버그 수정, 문서 작업처럼 성격이 명확히 다를 때만 커밋을 분리합니다.
- 변경하지 않은 파일이나 범위는 굳이 언급하지 않습니다.
- 반복적인 기본 검증은 매번 기계적으로 적지 않습니다.
- 중요한 테스트나 의미 있는 검증을 수행했다면 커밋 본문에 남깁니다.

권장 형식:

```text
커밋 제목

- 변경한 내용
- 변경한 이유나 영향
- 검증하거나 확인한 내용
```
