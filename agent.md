# Agent Rules

## 커밋 메시지 규칙

- 커밋 메시지는 반드시 한국어로 작성합니다.
- 커밋 내용은 한 문장으로 뭉뚱그리지 말고, 변경 범위별로 세분화해서 작성합니다.
- 기능 추가, 디자인 변경, 버그 수정, fix, 문서 작성처럼 변경 성격이 다르면 각각 별도 커밋으로 분리합니다.
- 각 커밋은 반드시 제목과 상세 내용을 함께 작성합니다.
- 제목은 해당 커밋의 핵심 목적을 짧게 요약합니다.
- 상세 내용에는 실제 변경 내용을 항목별로 나눠 적습니다.
- 검증한 내용이 있다면 상세 내용에 함께 적습니다.

권장 형식:

```text
커밋 제목

- 변경한 내용
- 변경한 이유나 영향
- 검증하거나 확인한 내용
```
